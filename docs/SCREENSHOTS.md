# Screenshots

> [!WARNING]
> **There are no screenshots, because there is no UI.**
>
> PropFix is being rebuilt from scratch. `src/` contains no shipped frontend
> code, `propfix --demo` does not exist, and `docs/screenshots/` is empty. No
> mockup, render, or placeholder image is presented anywhere in this repository
> as if it were a running application.
>
> This chapter is the **plan**: the shot list, and how regeneration will work
> once there is something to capture.

## Why this file exists now

The VulOS product standard requires UI products to ship a screenshot gallery
generated by a real screenshotter running against a real demo mode — and is
explicit that pure-backend repos should **not fake screenshots**. PropFix will be
a UI product, so it owes a gallery. Writing the shot list before the UI exists
means the demo dataset is designed to show the product's actual argument rather
than whatever happened to be seeded.

## The intended tooling

| Piece | Status |
|---|---|
| `propfix --demo` — in-memory seeded dataset, no DB, no config | 📐 Designed |
| `scripts/screenshots.mjs` — Playwright/Chromium | 📐 Designed, not written |
| `npm run screenshots` | 📐 Designed, not wired |
| `docs/screenshots/*.png` | Empty |
| README gallery | Placeholder section, explicitly marked |

Requirements on the screenshotter, from the standard and from the sibling repos
that got it right:

- **Playwright (Chromium)**, retina scale.
- Runs against **seeded demo data** so it needs **no real backend, no database,
  and no credentials**. If a new contributor cannot regenerate the gallery on a
  clean clone with two commands, it is broken.
- Outputs to `docs/screenshots/`.
- Captures light **and** dark where both are shipped.
- Wired into CI if CI exists, so a screen that stops rendering is a failing
  build rather than a stale PNG.

## The shot list

Ordered by what a first-time reader needs to understand, not by navigation
order. Each will be captured against demo data.

| # | Shot | What it must show |
|---|---|---|
| 1 | **Hero — building overview** | A portfolio of buildings with open job counts and spend. The one image that has to say "this is property maintenance software". |
| 2 | **Unit detail** | Units as **real rows** — the normalised `key` alongside the display `label` — with their job and inspection history. This is the fix for the legacy free-text-unit defect and it has to be visible. |
| 3 | **Job detail** | The append-only event thread with the `public` / internal visibility distinction rendered clearly, since that flag is a security control. |
| 4 | **Job costing** | Multiple `cost_entry` rows summing to the job total, including a **negative correction entry**, so the append-only model is legible at a glance. |
| 5 | **Inspection template editor** | Areas and items; per-item photo and comment requirements. |
| 6 | **Inspection capture** | Mid-run, on a tablet viewport, **with the offline indicator showing** — capture is designed for a basement with no signal and the screenshot should say so. |
| 7 | **Ingoing/outgoing comparison** | The differentiator. Item-by-item deltas including at least one **"not captured ingoing"** row, because the honest empty case is the point of the feature. |
| 8 | **Reporting** | Spend per building and per unit — the analysis the unit-as-a-row change exists to make correct. |
| 9 | **Sync / peers** | Manually enrolled peers, last round result, and the folder transport path. No discovery, visibly. |
| 10 | **Map** | Buildings positioned by `lat`/`lon` on MapLibre + Protomaps tiles — **no API key**, offline-capable. |

## Demo data requirements

The demo dataset is not filler; it has to demonstrate specific claims:

- At least one unit whose label would have collided under free-text matching
  (`Flat 3A` / `3A`), showing they resolve to one unit.
- A job with cost entries from **two different authors**, plus a negative
  correction, so the sum is visibly not last-writer-wins.
- A completed ingoing inspection **and** a matching outgoing one, with at least
  one deteriorated item, one unchanged item, and one item missing from the
  ingoing run.
- At least two peers, one of which is a folder/USB transport rather than a URL.
- Buildings with plausible coordinates so the map is not empty.

All of it fictional. Real tenant names, real addresses and real photographs of
real homes must never appear in a committed screenshot.
